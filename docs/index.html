设备控制对接

设备列表推送

设备列表分两部分，一部分是sdk返回的设备列表，一部分是api返回的。

这里分了两个modal，存储两份设备列表。

两个列表会有一些交互，设备上下线的时候sdk会推送，在： router/App.js中搜索registerDeviceListNotifications

getDeviceListSuccess = (data) => {
    const { dispatch } = this.props;
    if (data.devices) {
      dispatch({
        type: 'deviceList/filterDeviceListAndUpdate',
        payload: data.devices,
      });
      // 推送一份到api设备列表，更新在线状态
      dispatch({
        type: 'apiDeviceList/updateGizwitsDeviceList',
        payload: data.devices,
      });
    }
}

这里做了两件事，第一件是推送更新到deviceList modal，另一件是推送更新到apiDeviceList modal（用来更新设备的在线状态，以及相关的设备数据点，设备列表有时候会显示设备的状态，例如TDS之类的值，可以从sdk拿）。

router/App.js中搜索getDeviceStatusSuccess

如果有些设备列表显示了设备的数据点，可以从这里再推送一条消息去api devicelist modal更新数据。

关于apiDeviceList modal

sdk交互需要机智云的token，租赁后台获取token的方式是用sno换。

这边初步的处理方式是：

const data = yield call(queryUsingDevice);
if (data.success) {

// 获取设备列表

const deviceList = data.data.data.records;
payload.success && payload.success(deviceList);

if (deviceList.length > 0) {
    /**
     * 判断sdk是否已经有token
     * 如果没有，则用sno申请一个token
     */
  const gizwitsSdk = yield select(state => state.gizwitsSdk);
  if (!gizwitsSdk.token) {
    // 申请token
    const gizwitsTokenData = yield call(getGizwitsToken, deviceList[0].sno);
    console.log(gizwitsTokenData);
    const { gizAppId, token, uid } = gizwitsTokenData.data.data;
    // 把相关信息保存到本地
    localStorage.setItem('gizwitsUid', uid);
    localStorage.setItem('gizwitsToken', token);
    // 更新token等相关信息到modal
    yield put({
      type: 'gizwitsSdk/update',
      payload: {
        appId: gizAppId,
        token,
        uid,
      },
    });
    // 设置一个timeout，由于这个时候sdk不一定初始化完成（sdk初始化完成的回调函数不正确，这是个bug）
    // timeout函数见apiDeviceList modal
    yield call(timeout);

    // 调用获取设备列表 传递token过去
    yield put({
      type: 'deviceList/getList',
      payload: {
        backgroundLoad: true,
      },
    });
  }
}

这里完成了获取api设备列表的同时，把token传递给sdk，sdk开始上报数据。

设备控制的处理

super-app内置了设备控制的相关处理。

在container/DeviceList.js里，点击item跳到下一页的操作中，需要做几件事情。

  onClick = (data) => {
    const deviceData = JSON.parse(JSON.stringify(data));
    const { dispatch, deviceList } = this.props;
    // 设置当前激活的mac 如果是分组，则另外处理
    let activeData = null;
    console.log('deviceData', deviceData);
    console.log('deviceList.data', deviceList.data);
    // 通过mac，查找机智云的设备对象，确认设备已经链接
    deviceList.data.map((item) => {
      if (item.did === deviceData.gizDid) {
        activeData = {
          name: deviceData.name,
          productKey: item.productKey,
          did: deviceData.gizDid,
          mac: deviceData.mac,
        };
      }
    });

    if (activeData && deviceData.onlineStatus === 1) {
      dispatch({
        type: 'templateState/initStart',
      });
      router.go(`#/menu/devicePage/${activeData.productKey}/${activeData.mac}/${activeData.did}`);
      /** 标记当前激活设备 */
      dispatch({
        type: 'deviceList/updateActiveDevice',
        payload: {
          did: activeData.did,
          productKey: activeData.productKey,
          name: activeData.name,
        },
      });
    } else {
      // 提示错误
      Toast.info('设备不在线，请检查设备是否正常运行。');
    }
  }

第一步是比对两组设备列表，获取相关的信息（name,productKey,did,mac），如果api deviceList有带这些参数的话就不用对比了。这边还需要判断设备的状态，如果设备离线，则不允许跳转过去。

第二步是跳转到页面，并更新当前激活的设备，然后sdk就会开始推送数据到模版了。
